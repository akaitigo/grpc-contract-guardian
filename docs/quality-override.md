# 品質チェックリスト追加分 — CLIツール

> Layer-0（共通）+ Layer-1（言語別）のチェックリストに**追加**する項目のみ。

## CLI固有の品質基準

### ユーザビリティ
- [ ] `--help` フラグが全コマンド/サブコマンドで動作する
- [ ] `--version` フラグでバージョンが表示される
- [ ] 引数不足時に使い方（usage）が表示される
- [ ] エラーメッセージが stderr に出力される（stdout は正常出力のみ）
- [ ] 終了コード: 成功=0, ユーザーエラー=1, 内部エラー=2

### 入出力
- [ ] stdin からのパイプ入力に対応している（該当する場合）
- [ ] 出力が `--json` や `--quiet` フラグで制御できる（該当する場合）
- [ ] カラー出力は `NO_COLOR` 環境変数で無効化できる
- [ ] 長時間処理はプログレス表示がある

### E2Eテスト
- [ ] bats-core テストが全主要コマンドをカバーしている
- [ ] 正常系・異常系（不正引数、存在しないファイル等）の両方をテスト
- [ ] CI環境でbatsテストが実行される設定がある

### 配布
- [ ] バイナリ名がプロジェクト名と一致している
- [ ] クロスプラットフォームビルドの確認（Linux, macOS, Windows）
- [ ] Makefile に `install` ターゲットがある
