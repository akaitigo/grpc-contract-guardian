# Hooks 設計ガイド（Layer-0: 全プロジェクト共通）

## 3層Hook構造

Hooksは「決定論的ツールで品質を強制する」ための仕組み。
LLMに判断させず、ルールベースで即座にフィードバックする。

---

### PreToolUse（安全ゲート）

ツール実行**前**にブロック判定。不可逆な操作を未然に防ぐ。

| ガード対象 | matcher | 例 |
|------------|---------|-----|
| リンター設定の改竄 | `Edit\|Write\|MultiEdit` | `.golangci.yml`, `Makefile` 等 |
| 破壊的コマンド | `Bash` | `rm -rf`, `git push --force`, `git reset --hard` |
| 機密ファイル | `Edit\|Write\|MultiEdit` | `.env`, `credentials.*`, `*.pem`, `*.key` |
| ロックファイル直接編集 | `Edit\|Write\|MultiEdit` | `go.sum` |

### PostToolUse（品質ループ）

ファイル変更**直後**に自動で品質チェック＋修正を実行。

| アクション | タイミング |
|------------|-----------|
| フォーマッタ適用 | ファイル保存後（自動修正 → サイレント） |
| リンター自動修正 | フォーマット後（`--fix` → サイレント） |
| 残存違反のJSON返却 | 自動修正不能なエラーを `additionalContext` で返す |

### PreCompact（コンテキスト保護）

コンテキスト圧縮**前**に状態ファイルをバックアップ。

### Stop（完了ゲート）

`make check` で全条件を満たすまで完了させない。
CLIツールは追加で `bats test/*.bats` を実行。

---

## フィードバック速度階層

```
速度      仕組み                    検出率           対応範囲
──────────────────────────────────────────────────────────────────
ms     PostToolUse Hook           40-50%自動解決    フォーマット、lint自動修正
s      プリコミット (Lefthook)     30-40%検出        テスト実行、lint残存違反
min    CI (GitHub Actions)        最終ゲート         テスト回帰、ビルド壊れ
hour   人間レビュー                戦略的判断のみ     アーキテクチャ判断
```
